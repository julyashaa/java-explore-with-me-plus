# дополнительная функциональность - комментарии к событиям

предоставляет REST API для работы с комментариями к событиям.  
Реализована **трёхуровневая модель доступа**:

- **Публичный** – просмотр комментариев без авторизации.
- **Приватный** – создание, редактирование, удаление и просмотр **своих** комментариев (только для автора).
- **Административный** – расширенный поиск, модерация (редактирование/удаление любых комментариев).

---

##  Эндпоинты

### 1. Публичный API – `PublicCommentController`

| Метод | Эндпоинт                     | Описание                   | Параметры запроса | Тело | Ответ                   | Коды ошибок               |
|-------|------------------------------|----------------------------|-------------------|------|-------------------------|---------------------------|
| GET   | `/comments/{commentId}`      | Получить комментарий по ID | –                 | –    | `ShortCommentDto`       | 404 – комментарий не найден |
| GET   | `/events/{eventId}/comments` | Комментарии события        | `from`, `size`    | –    | `List<ShortCommentDto>` | 404 – событие не найдено   |

**Особенности:**
- Проверяется существование события; при отсутствии – `404 Not Found`.
- Публичное DTO **не содержит автора** (может быть расширено при необходимости).

---

### 2. Приватный API – `PrivateCommentController`

| Метод  | Эндпоинт                               | Описание                            | Параметры запроса | Тело               | Ответ              | Коды ошибок                                   |
|--------|----------------------------------------|-------------------------------------|-------------------|--------------------|--------------------|-----------------------------------------------|
| POST   | `/users/{userId}/comments`             | Создать комментарий                 | –                 | `NewCommentDto`    | `CommentDto` (201) | 400 – невалидные данные<br>404 – пользователь/событие не найдены |
| GET    | `/users/{userId}/comments/{commentId}` | Получить свой комментарий (с автором)| –                | –                  | `CommentDto`       | 403 – не автор<br>404 – не найдено            |
| GET    | `/users/{userId}/comments`             | Все комментарии пользователя        | `from`, `size`    | –                  | `List<CommentDto>` | 404 – пользователь не найден                 |
| PATCH  | `/users/{userId}/comments/{commentId}` | Обновить текст комментария          | –                 | `UpdateCommentDto` | `CommentDto`       | 403 – не автор<br>404 – не найдено            |
| DELETE | `/users/{userId}/comments/{commentId}` | Удалить комментарий                | –                 | –                  | `204 No Content`   | 403 – не автор<br>404 – не найдено            |

**Ключевые правила:**
- **Только автор** может редактировать или удалять комментарий.
---

### 3. Административный API – `AdminCommentController`

| Метод  | Эндпоинт                      | Описание                               | Параметры запроса                                                       | Тело               | Ответ              | Коды ошибок |
|--------|-------------------------------|----------------------------------------|-------------------------------------------------------------------------|--------------------|--------------------|-------------|
| GET    | `/admin/comments`             | Поиск комментариев с фильтрацией       | `ids`, `userId`, `eventId`, `rangeStart`, `rangeEnd`, `from`, `size`    | –                  | `List<CommentDto>` | 400 – невалидные параметры |
| PATCH  | `/admin/comments/{commentId}` | Обновить любой комментарий             | –                                                                       | `UpdateCommentDto` | `CommentDto`       | 404 – комментарий не найден |
| DELETE | `/admin/comments/{commentId}` | Удалить любой комментарий              | –                                                                       | –                  | `204 No Content`   | 404 – комментарий не найден |

**Особенности:**
- Администратор может изменять/удалять любой комментарий.
- **фильтрация** :
    - `ids` – список идентификаторов комментариев.
    - `userId` – автор комментария.
    - `eventId` – событие.
    - `rangeStart` / `rangeEnd` – диапазон дат создания.
- Валидация дат: если переданы оба параметра, `rangeStart` не может быть позже `rangeEnd` (аннотация `@ValidDateRange`).

##  DTO

| DTO                        | Назначение                                  | Ключевые поля                                                                       |
|----------------------------|---------------------------------------------|--------------------------------------------------------------------------------------|
| `NewCommentDto`            | Создание комментария                       | `eventId` (Long), `text` (String, 1‑2000)                             |
| `UpdateCommentDto`         | Обновление текста                          | `text` (String, до 2000)                                                |
| `CommentDto`               | Полное представление (приват/админ)        | `id`, `eventId`, `text`, `author` (UserShortDto), `createdOn`, `editedOn`           |
| `ShortCommentDto`          | Краткое представление (публичное)          | `id`, `eventId`, `text`                                                             |
| `GetCommentsDtoParams`     | Пагинация для публичных/приватных списков  | `from` (default 0), `size` (default 10)                                             |
| `GetCommentsAdminDtoParams`| Пагинация + фильтры для администратора     | `ids`, `userId`, `eventId`, `rangeStart`, `rangeEnd`, `from`, `size`                |
